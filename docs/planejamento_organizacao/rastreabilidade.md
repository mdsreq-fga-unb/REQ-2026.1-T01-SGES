# Rastreabilidade como Memória de Sentido

A rastreabilidade no Miro/Figma foi estruturada como uma **"memória de sentido"**, conectando a estratégia de negócio à execução técnica através dos seguintes níveis:

## Objetivos Específicos

- Estabelecem o **"porquê"** do sistema
- Ancoram o valor de negócio

## Características do Produto

- Funcionam como a ponte que traduz os objetivos em capacidades gerais do software

## Requisitos Funcionais

- Detalham o comportamento observável necessário para realizar cada característica

## Casos de Uso

- Representam visualmente as interações entre atores e sistema
- Garantem que o requisito faça sentido no contexto de uso
